# How to Make a CSV File and Upload It to Itemcount Using Import CSV

## <mark style="color:orange;">**How to make CSV file.**</mark>

1. Go to inventory items.&#x20;

<figure><img src="../../.gitbook/assets/Go to inventory items..png" alt=""><figcaption></figcaption></figure>

2. Click on Import CSV at the top of the right side.&#x20;

<figure><img src="../../.gitbook/assets/Click on Import CSV at the top of the right side..png" alt=""><figcaption></figcaption></figure>

3. This is the details page of the csv file; some fields are required, while others are optional.&#x20;

<figure><img src="../../.gitbook/assets/This is the details page of the csv file; some fields are required, while others are optional..png" alt=""><figcaption></figcaption></figure>

4. Click on Download example to get your own example for your guide.&#x20;

<figure><img src="../../.gitbook/assets/Click on Download example to get your own example for your guide..png" alt=""><figcaption></figcaption></figure>

5. This is the data you may obtain after downloading the sample, which is a csv file.&#x20;

{% hint style="info" %}
NOTE: The following areas must be filled out: **SKU, name, type, category, count, unit, market item?, and should track?** Not stated are optional.
{% endhint %}

<figure><img src="../../.gitbook/assets/This is the data you may obtain after downloading the sample, which is a csv file..png" alt=""><figcaption></figcaption></figure>

6. Stock-keeping unit (SKU), item alias, and name can all be the same, but to avoid errors, design a sku that combines letters and numbers so that it does not contradict your existing goods.&#x20;

{% hint style="info" %}
Note: You can leave the Item Alias field blank or make it the same as the item name or sku.
{% endhint %}

<figure><img src="../../.gitbook/assets/Stock-keeping unit (SKU), item alias, and name can all be the same, but to avoid errors, design a sku that combines letters and numbers so that it does not contradict your existing goods..png" alt=""><figcaption></figcaption></figure>

7. Type: "finished-goods" "raw-material" "consumable" "service-and-repair" You have the option of selecting the type of item.

{% hint style="info" %}
Note: Every type name must begin with a small letter.
{% endhint %}

<figure><img src="../../.gitbook/assets/You have the option of selecting the type of item..png" alt=""><figcaption></figcaption></figure>

8. In each item, you may create your own category. It will be added to your category list.&#x20;

<figure><img src="../../.gitbook/assets/In each item, you may create your own category. It will be added to your category list..png" alt=""><figcaption></figcaption></figure>

9. You can enter the cost and price of the item, but if you don't know, simply input zero (0).

{% hint style="info" %}
Remember that leaving the field blank will result in an error, therefore enter Zero (0). and Numbers should not be separated by commas (,) to avoid error.
{% endhint %}

<figure><img src="../../.gitbook/assets/You can enter the cost and price of the item, but if you don&#x27;t know, simply input zero.png" alt=""><figcaption></figcaption></figure>

10. The term count refers to the overall quantity of the product. If it doesn't have quantity, put Zero (0).&#x20;

{% hint style="info" %}
Remember that leaving the field blank will result in an error, therefore enter Zero (0)
{% endhint %}

<figure><img src="../../.gitbook/assets/The term count refers to the overall quantity of the product. If it doesn&#x27;t have quantity, put Zero.png" alt=""><figcaption></figcaption></figure>

11. It is the unit of measurement for an item. Simply put the abbreviation. (click on this link to see the unit of measurement)&#x20;

{% hint style="info" %}
It should be noted that not abbreviating a unit of measurement might lead to errors.
{% endhint %}

<figure><img src="../../.gitbook/assets/It is the unit of measurement for an item. Simply put the abbreviation..png" alt=""><figcaption></figcaption></figure>

12. Market item signifies it will be sold and should be tracked to see whether it can add or deplete item quantity. Simply type YES or NO.&#x20;

{% hint style="info" %}
A reminder that the letters YES and NO can be capitalized.
{% endhint %}

<figure><img src="../../.gitbook/assets/Market item signifies it will be sold and should be tracked to see whether it can add or deplete item quantity. Simply type YES or NO..png" alt=""><figcaption></figcaption></figure>

13. This is optional; you can provide the item's barcode or image link.

<figure><img src="../../.gitbook/assets/This is optional; you can provide the item&#x27;s barcode or image link..png" alt=""><figcaption></figcaption></figure>

<figure><img src="../../.gitbook/assets/Upload your completed csv file here..png" alt="" width="563"><figcaption></figcaption></figure>

22. This is the column in your csv with an issue. Once corrected, return to the csv upload.

24. Importing your data.

<figure><img src="../../.gitbook/assets/Importing your data..png" alt="" width="563"><figcaption></figcaption></figure>

25. It will occur if there is a missing field or an inaccurate term in the csv, therefore double-check your csv.&#x20;
